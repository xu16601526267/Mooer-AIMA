## Step 1: Kimi API 连通性 ✅

**动作**: curl 从 linux-1 调用 Kimi chat completions API
**命令**:
```bash
curl -s https://api.kimi.com/coding/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-kimi-***" \
  -H "User-Agent: claude-code/2.1.5" \
```

**结果**: ✅ HTTP 200, 返回 OpenAI 兼容格式
**发现**:
- dev-mac 无法访问 (SSL error, GFW)
- linux-1 正常访问，延迟约 2s
- 响应含 `reasoning_content` 字段（Kimi 特有，不影响标准解析）
- model 返回值: `kimi-for-coding` ✓

## Step 9: Edge LLM 配置 + Explorer Tier 检测 ✅

**动作 9a**: 配置 Edge 端 LLM 指向 Kimi
```bash
./aima config set llm.endpoint https://api.kimi.com/coding/v1
./aima config set llm.model kimi-for-coding

./aima config set llm.user_agent claude-code/2.1.5
```
**结果**: ✅ 所有配置写入 SQLite，每次 set 触发 LLM hot-reload
**日志**: `INFO LLM settings hot-reloaded endpoint=https://api.kimi.com/coding/v1 model=kimi-for-coding has_api_key=true`

**动作 9b**: 验证 Agent LLM 连通性
```bash
./aima ask "say hi in 3 words"
```
**结果**: ✅ `"Hi there, friend!"` — Kimi 通过 Edge Agent 通道响应
**Agent Status**: `agent_available=true, agent_tool_mode=unknown`

**动作 9c**: Tier 检测逻辑
- CLI 模式每次都是新进程，tool mode 初始为 `unknown` → 始终 Tier 1
- 需通过 `aima serve` 长驻进程，先 `agent.ask` 触发 tool mode 检测，再看 tier 升级

**发现**:
- Edge LLM env 变量名是 `AIMA_API_KEY` (非 `AIMA_LLM_API_KEY`)，SQLite config key 是 `llm.api_key`
- Tier 升级需要 `Agent.Ask()` 调用（内含 tool mode 自动检测），LLMPlanner 直接调 `llm.ChatCompletion()` 不触发检测
- **设计缺陷 (O4)**: Explorer 无法自行从 Tier 1 升级到 Tier 2，依赖外部 agent 调用
