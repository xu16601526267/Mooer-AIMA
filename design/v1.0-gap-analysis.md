### Key Findings

1. **L0-L2 foundation is solid.** Hardware detection (8 GPU vendors), multi-runtime (K3S/Docker/Native),
   knowledge-driven config resolution, and 56 MCP tools are production-grade.
2. **L3 Agent has scaffolding.** The Go Agent (L3a) has a working tool-calling loop. Patrol, tuning,
   and self-healing MCP tools are wired but need real Agent orchestration to exercise fully.
   External Agents connect via MCP for complex workflows.
3. **Knowledge query engine is complete.** All 6 query types (Search/Compare/Gaps/Similar/Lineage/Aggregate)
   are implemented with SQL-backed operations.
4. **YAML "dark data" partially activated (v0.0.2).** `cold_start_s`, `startup_time_s`,
   `cold_start_time_s`, and `tdp_watts` are now surfaced in `ResolvedConfig` and used by
   `CheckFit()` for power budget warnings and dry-run resource estimates. Remaining unused:
   `compute_units`, `bandwidth_gbps`.
5. **Central knowledge server operational (v0.0.2).** `cmd/central` — SQLite-backed REST service.
   Edge devices push/pull via `knowledge.sync_push/pull` MCP tools.
6. **No tunnel or remote access.** Fleet works over LAN (mDNS), but no WAN connectivity.

## 2. Layered Design Review

PRD defines a multi-layer intelligence stack (L0-L3a). Understanding which gaps belong to which layer
clarifies implementation approach:

```
Layer    │ Scope                           │ Implementation   │ Gap Status
─────────┼─────────────────────────────────┼──────────────────┼───────────
L0       │ Hardcoded defaults (YAML)       │ Go code          │ COMPLETE
L1       │ Human CLI overrides             │ Go code          │ COMPLETE
L2a      │ Engine Asset matching           │ Go code          │ COMPLETE
L2b      │ Model Asset variant selection   │ Go code          │ COMPLETE
L2c      │ Tuning optimal (from benchmark) │ Go code + Agent  │ COMPLETE — WithGoldenConfig() injected into Resolve() between L0 and L1
L3a      │ Go Agent (tool-calling loop)    │ Go code          │ PARTIAL — reactive + patrol/tuning/healing with auto-diagnosis
```

**Gap distribution by layer:**
- L0-L2b gaps (Go deterministic logic): S6, D3 amplifier eval, I5
- L2c-L3 gaps (Agent dynamic optimization): Full A2/A3/A4/A5 orchestration
- Infrastructure gaps (new services): Tunnel (WAN connectivity)
